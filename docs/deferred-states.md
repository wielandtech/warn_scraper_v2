# Deferred state scrapers

States from the V1 active set whose live sources have drifted in ways that
need investigation (URL moved, new bot protection, page rebuilt with
JS-rendered content). Listed here so the Phase-4-style enrichment / heal
agent has a queue to work through later.

Last checked: 2026-05-22.

| State | V1 URL | What changed | Next step |
|-------|--------|--------------|-----------|
| **CT** | `ctdol.state.ct.us/progsupt/bussrvce/warnreports/warn{year}.htm` | **Resolved** — site migrated to `dolpublicdocumentlibrary.ct.gov`. Scraper hits `GetPagedBlobs` JSON API (Azure Blob Storage). Parses employer/city/date from PDF filenames; falls back to `modifiedDate`. Implemented in `warn_v2/scrapers/states/ct.py`. | — |
| **GA** | `https://www.dol.state.ga.us/public/es/warn/searchwarns` | WARN administration transferred from GDOL to **TCSG** (Technical College System of Georgia) as of Jan 2023. Public listing at `tcsg.edu/warn-public-view/` — confirmed live, table is JS-rendered (no static rows in a plain fetch). Old GDOL form still live but frozen at June 2013. | Requires browser automation (Playwright). Target: `https://www.tcsg.edu/warn-public-view/`. Contact: `rapidresponse@worksourcega.org`. |
| **MI** | `https://milmi.org/warn` | Confirmed JS-rendered. New URL: `michigan.gov/leo/bureaus-agencies/wd/data-public-notices/warn-notices`. Uses Sitecore XA faceted search (`facet-warn---counties` component). Sitecore XA Searcher API (`/api/sitecore/Searcher/Search`) returns a "Site Maintenance" HTML page — no accessible JSON endpoint found. No downloadable data files on the page. | Requires browser automation (Playwright). Monitor for a public data API or CSV download to appear. |
| **NJ JSP** | `lwd.state.nj.us/WorkForceDirectory/warn.jsp?...` | Behind Incapsula (403 + JS challenge). V2 falls back to the still-live yearly PDF at `nj.gov/labor/assets/PDFs/WARN/{year}_WARN_Notice_Archive.pdf` (see `warn_v2/scrapers/states/nj.py`). | If the PDF ever stops being published, scraping the JSP page would need a real browser (Playwright) — not worth it. |
| **OK** | `okjobmatch.com/search/warn_lookups?...` | Migrated to **Salesforce Experience Cloud** at `employoklahoma.gov/Participants/s/warnnotices` (org: `oescoowd.my.salesforce.com`, Launchpad package). Aura endpoint (`/s/sfsites/aura`) is live but requires a csrfV2 `aura.token` generated by JavaScript at page load — all data actions return empty without it. No Apex REST endpoints or static data in HTML. Direct API access confirmed impossible without a headless browser. | Requires Playwright: load page, collect `eikoocnekot` cookie / `aura.token`, intercept the `apex://` Aura action that returns WARN records. |
| **OH** | `jfs.ohio.gov/warn/current.stm` | Confirmed JS-rendered. Page renders via IBM WebSphere Portal with no static table content. Checked `data.jfs.ohio.gov` — no WARN dataset published there. | **PA** | `dli.pa.gov/Individuals/Workforce-Development/warn/notices/Pages/{Month}-{Year}.aspx` | **Resolved** — new URL confirmed live at `pa.gov/agencies/dli/programs-services/workforce-development-home/warn-requirements/warn-notices`. AEM accordion page, plain HTML, no JS required. 300 accordion items covering 2023-present. Implemented in `warn_v2/scrapers/states/pa.py`. | — |
| **OR** | `ccwd.hecc.oregon.gov/Layoff/WARN` | **Resolved** — county filter (`?County=Name&page=N`) restricts to OR only. Scraper iterates all 36 counties with pagination. Implemented in `warn_v2/scrapers/states/or_.py`. | — |
| **IA** | `iowawf.gov/147/WARN-Notices` | **Resolved** — page shows JS Tableau visualization, but has a stable ADA-compliant Excel download at `workforce.iowa.gov/media/3025/download?inline`. Scraper fetches cumulative XLSX directly. Implemented in `warn_v2/scrapers/states/ia.py`. | — |

| **MA** | `mass.gov/lists/...warn-act-layoff-list` | CDN returns 403 "Not allowed" for all bot requests, including the weekly CSV download URLs (`/doc/warn-report-for-week-ending-{date}/download`). `eolwd.mass.gov` hostname no longer resolves. | Requires a browser with real TLS fingerprint (Playwright/Camoufox). | **AR** | *(no V1 scraper)* | WARN notices are treated as confidential in Arkansas; no public disclosure required or published. | No action — state law prohibits public release. |
| **NH** | *(no V1 scraper)* | New Hampshire does not require public posting of WARN notices; the state agency does not publish a list. | No action — no public data source. |
| **WY** | *(no V1 scraper)* | Wyoming has no state WARN law and does not publish federal WARN filings. | No action. |
| **WV** | *(no V1 scraper)* | Per-notice PDFs available individually on request; no machine-readable list published. | Would require fetching index + per-notice PDF parsing. Low value. |

These are good candidates for the heal agent in "greenfield" mode (Phase 4-ish)
— given a live sample and a contract, it can write a scraper from scratch.
For now they fail at `fetch()` and the runner logs `fetch_failed` cleanly.
